## 1. Metrics Monitored

### Metadata Completeness
- Percentage of required frontmatter fields filled (across all docs)
- Required Field Debt (count of missing required fields total)
- Missing fields per taxonomy dimension: `device`, `role`, `useCase`, `skillLevel`, `productName`
- Docs with any missing required field (count + file list)

### Schema Intelligence
- **Guessed Fields** — total count of auto-generated frontmatter values
- **Docs with Guesses** — count of docs where any field was auto-guessed
- **Fully Authored** — count of docs where all fields are human-written
- Field Coverage per field name (% of docs that have the field populated)
- **Schema Completion Percentage** *(owned by Librarian)* — `(fully authored docs / total docs) × 100`

### Content Quality
- Docs with Placeholders (count + file list)
- Missing Titles (count)
- Missing Descriptions (count)
- Missing Keywords (count)
- Placeholders by Field (breakdown)
- Docs Needing Attention (composite list)

### Broken Links & Images *(moved from Gatekeeper)*
- Broken Internal Links (count + affected docs + source→target mapping)
- Broken Image References (count + affected file list)
- Link/image remediation proposals (AUTO_REMEDIATE via PR)

### DITA Semantic Conversion *(structural/schema concern)*
- Migration Coverage % (files using DITA semantic components vs. total task files)
- Files with DITA Components (count + file list)
- Legacy DITA XML Tags remaining (count + file list)
- Task Files without Semantic Wrappers (count)
- Component Usage breakdown (Prereq, TaskResult, UIControl, MenuCascade, KBD, TaskContext, StepResult)

### Semantic Loss from DITA→Markdown Conversion *(content integrity)*
All docs in this repository were converted from DITA XML to Markdown/MDX via automated scripts. The conversion introduces predictable semantic loss patterns that Librarian tracks and flags for remediation.

**Tracked Loss Patterns (DL-01 through DL-10):**

| Test ID | Pattern | Severity | Description |
|---------|---------|----------|-------------|
| DL-01 | Flattened Tables | P1 | DITA `<table>` / `<simpletable>` elements rendered as bare text lines with no pipe delimiters. The most prevalent issue — table structure is completely destroyed. |
| DL-02 | Missing Admonitions | P2 | DITA `<note type="caution|warning|danger|tip">` elements converted to plain paragraphs instead of `:::caution` / `:::warning` blocks. |
| DL-03 | Duplicate List Content | P2 | Lists inside DITA table cells extracted twice — once as bullet list, once as bare text. Converter artifact from cell-to-text fallback. |
| DL-04 | Gutted/Empty Body | P1 | Docs where body content was lost during conversion. Often marked with `description: Recovered content. Manual restoration in progress.` sentinel. |
| DL-05 | Orphaned HTML Entities | P3 | HTML entities like `&lt;` / `&gt;` / `&amp;` surviving from DITA source that should be literal characters in Markdown. |
| DL-06 | Unclosed Code Fences | P2 | Code blocks opened with ` ``` ` but never closed. Often caused by merged DITA `<codeblock>` elements. |
| DL-07 | Empty Headings | P2 | Section headings (`##`) with no content beneath them before the next heading. Indicates lost body content under that section. |
| DL-08 | Orphaned DITA/HTML Tags | P1 | Raw XML tags like `<p>`, `<ul>`, `<table>`, `<prereq>`, `<uicontrol>` etc. still present in the Markdown outside code blocks. |
| DL-09 | Title Echoes | P3 | Frontmatter `title` duplicated as plain text or H1 in the first few body lines. Docusaurus already renders the title from frontmatter. |
| DL-10 | Broken Procedures | P2 | Numbered steps converted to bullet lists. Items starting with "Step N", "First,", "Then," etc. as bullets instead of ordered list items. |

**Remediation priority:** DL-01 (flattened tables) and DL-04 (gutted bodies) are the highest-impact issues. DL-01 affects readability and usability of specification/reference docs. DL-04 means entire pages have no meaningful content.

**Data source:** `static/data/dita-loss-report.json` — generated by `scripts/dita-loss-scanner.mjs` during each build.

**Best Practices for Reformatting Converted Content:**
1. **Tables:** Reconstruct with proper Markdown pipe syntax. Use header row + separator row + data rows. For complex tables with lists in cells, use HTML `<table>` tags.
2. **Admonitions:** Wrap note/caution/warning text in `:::note` / `:::caution` / `:::warning` / `:::danger` blocks.
3. **Duplicate content:** Delete the plaintext duplicate — keep the bulleted list version.
4. **Empty bodies:** Flag for manual restoration from DITA source XML. These cannot be auto-fixed.
5. **Orphaned entities:** Replace `&lt;` → `<`, `&gt;` → `>`, `&amp;` → `&` outside code blocks.
6. **Unclosed fences:** Add closing ` ``` ` or split merged code blocks into separate fenced blocks with explanatory text between them.
7. **Empty headings:** Either add content beneath the heading or remove the heading if the section is irrelevant.
8. **Orphaned tags:** Convert to proper Markdown equivalents or DITA semantic components (`<UIControl>`, `<Prereq>`, etc.).
9. **Title echoes:** Delete the body line that duplicates the frontmatter title. AUTO_REMEDIATE candidates.
10. **Broken procedures:** Convert bullet lists to numbered lists (`1.`, `2.`, `3.`). Remove ordinal words ("Step 1", "First,") that become redundant.

## 2. Execution Logic

### Step 1 — Metadata Audit
Scan all docs for required frontmatter fields. Compute Required Field Debt. Separate guessed fields from fully authored fields. Compute Schema Completion Percentage.

### Step 2 — Schema Quality Review
Flag all docs where `guessedFields > 0`. Prioritize docs where guessed fields cover required taxonomy dimensions (`device`, `role`, `useCase`). These are high-risk for MCP server utility degradation.

### Step 3 — Content Quality Scan
Identify docs with placeholder text, missing titles, missing descriptions, missing keywords, or missing slugs.

### Step 4 — Broken Links & Images
Scan for broken internal links and broken image references (using ENG-04, ENG-05, ENG-09 data from build report). Propose auto-remediation PRs for broken images. Surface broken links as CLICK_TO_FIX.

### Step 5 — DITA Migration Tracking
Scan all docs for DITA semantic component usage (Prereq, TaskResult, UIControl, MenuCascade, KBD, TaskContext, StepResult). Identify task files without semantic wrappers. Detect legacy DITA XML tags. Compute migration coverage percentage. Surface low-coverage areas as CLICK_TO_FIX.

### Step 6 — MDX Churn Analysis
Count MDX file changes since last snapshot. If churn > threshold (default: 20 files), flag as P2.

### Step 7 — Publish Rate Check
Verify Published/Draft split. Flag if Publish Rate drops below 90% (configurable).

### Step 8 — Remediation
Apply remediations per the table in Section 3.

## 3. Remediation Actions

| Issue | Severity | Remediation Type | Output |
|-------|----------|-----------------|--------|
| Broken internal link | P1 | CLICK_TO_FIX | Dashboard link to file |
| Broken image reference | P1 | AUTO_REMEDIATE | GitHub PR to `docs/staging` |
| Required field missing (taxonomy: device/role/useCase) | P1 | AUTO_REMEDIATE | GitHub PR to `docs/staging` with proposed field value |
| High guessed-field count on required taxonomy fields | P1 | CLICK_TO_FIX | Dashboard flag + GitHub file link for human review |
| Docs with placeholder text | P1 | CLICK_TO_FIX | Dashboard flag + GitHub file link |
| MDX churn > threshold | P2 | ESCALATE | JIRA ticket |
| Missing title or slug | P2 | AUTO_REMEDIATE | GitHub PR to `docs/staging` with proposed value |
| Missing description or keywords | P2 | AUTO_REMEDIATE | GitHub PR to `docs/staging` with AI-generated proposal |
| DITA migration coverage < 50% | P2 | CLICK_TO_FIX | Dashboard flag with file list |
| Legacy DITA XML tags found | P2 | CLICK_TO_FIX | Dashboard flag with file list |
| Schema Completion % < 70% | P2 | ESCALATE | JIRA ticket |
| Draft docs not published (Publish Rate < 90%) | P2 | CLICK_TO_FIX | Dashboard flag with GitHub link |
| Low-priority guessed fields (non-taxonomy) | P3 | CLICK_TO_FIX | Dashboard annotation for human review |
| Flattened tables (DL-01) | P1 | CLICK_TO_FIX | Dashboard flag + file list in DITA Migration tile |
| Gutted/empty body content (DL-04) | P1 | CLICK_TO_FIX | Dashboard flag + file list (requires manual restoration) |
| Orphaned DITA/HTML tags (DL-08) | P1 | CLICK_TO_FIX | Dashboard flag with file + line reference |
| Duplicate list content (DL-03) | P2 | AUTO_REMEDIATE | GitHub PR to remove plaintext duplicates |
| Empty headings (DL-07) | P2 | CLICK_TO_FIX | Dashboard flag with file list |
| Missing admonitions (DL-02) | P2 | CLICK_TO_FIX | Dashboard flag + suggested :::note wrapper |
| Unclosed code fences (DL-06) | P2 | AUTO_REMEDIATE | GitHub PR to close fences |
| Broken procedures (DL-10) | P2 | CLICK_TO_FIX | Dashboard flag with file + line reference |
| Title echoes in body (DL-09) | P3 | AUTO_REMEDIATE | GitHub PR to remove duplicate title line |
| Orphaned HTML entities (DL-05) | P3 | AUTO_REMEDIATE | GitHub PR to replace entities with literal chars |

> **Note:** All `AUTO_REMEDIATE` actions are scoped to **metadata and frontmatter only** unless explicitly listed as body-safe (DL-03, DL-05, DL-06, DL-09 are body-safe auto-remediations). All commits target `docs/staging` and require one human reviewer before merge to `main`. Auto-generated values (descriptions, keywords) must be labeled as AI-generated in the PR body.
